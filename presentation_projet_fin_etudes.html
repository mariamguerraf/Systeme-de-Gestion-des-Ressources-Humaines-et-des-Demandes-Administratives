Gestion des Utilisateurs et Rôles

🔐 Système d'Authentification Multi-Rôles

Le système implémente un contrôle d'accès granulaire basé sur 4 rôles principaux avec des permissions spécifiques pour chaque type d'utilisateur.

👨‍💼 ADMIN
  • Gestion complète des enseignants
  • Gestion des fonctionnaires
  • Vue globale des statistiques
  • Administration système
👩‍💼 SECRÉTAIRE
  • Traitement des demandes
  • Approbation/Rejet
  • Téléchargement documents
  • Gestion commentaires
👨‍🏫 ENSEIGNANT
  • Gestion profil personnel
  • Soumission demandes
  • Attestations de travail
  • Ordres de mission
👨‍💼 FONCTIONNAIRE
  • Interface personnalisée
  • Demandes de congé
  • Ordres de mission
  • Suivi des demandes
Sécurité : Chaque interface est protégée par JWT avec vérification des permissions. L'accès aux données est filtré selon le rôle de l'utilisateur connecté.

Architecture Technique Détaillée

🏗️ Structure du Projet

Systeme-de-Gestion-RH/ ├── back_end/ │ ├── main.py # API FastAPI principale │ ├── models.py # Modèles SQLAlchemy │ ├── schemas.py # Schémas Pydantic │ ├── database.py # Configuration DB │ ├── routers/ # Endpoints par module │ │ ├── auth.py # Authentification │ │ ├── demandes.py # Gestion demandes │ │ ├── enseignant.py # API enseignants │ │ └── users.py # Gestion utilisateurs │ └── uploads/ # Stockage fichiers ├── src/ │ ├── components/ # Composants React │ ├── pages/ # Pages par rôle │ ├── contexts/ # État global │ └── services/ # Services API └── public/ # Assets statiques

🔧 Configuration Développement

Backend (Port 8000)
  • FastAPI avec auto-documentation
  • Hot reload avec Uvicorn
  • CORS configuré pour développement
Frontend (Port 8080)
  • Vite dev server avec HMR
  • Proxy API automatique
  • TypeScript strict mode

🛡️ Sécurité Implémentée

  • JWT Authentication avec expiration et refresh tokens
  • Validation des données côté client et serveur (Pydantic)
  • Contrôle d'accès basé sur les rôles (RBAC)
  • Upload sécurisé avec vérification des types MIME
  • Protection CSRF et validation des tokens

⚡ Performance et Optimisation

  • React Query pour la gestion du cache et des requêtes
  • Lazy loading des composants avec React.lazy
  • Pagination côté serveur pour les grandes listes
  • Compression des assets avec Vite
  • Code splitting automatique par route

Déploiement et Configuration

🖥️ Configuration Windows

Le projet est optimisé pour le développement sous Windows avec des scripts de démarrage automatisés :

start_app.bat

Lance automatiquement le backend et le frontend en parallèle

Scripts individuels

start_backend.bat et start_frontend.bat pour un contrôle granulaire

🌐 URLs d'Accès

  • Frontend : http://localhost:8080
  • Backend API : http://localhost:8000
  • Documentation API : http://localhost:8000/docs (Swagger UI)
  • Alternative Doc : http://localhost:8000/redoc

📦 Installation et Lancement

  1. Cloner le projet et naviguer dans le dossier
  2. Backend : pip install -r back_end/requirements.txt
  3. Frontend : npm install
  4. Lancement : Double-clic sur start_app.bat

🚀 Recommandations Production

🌐 Endpoints API Principaux

POST /auth/login - Authentification utilisateur
GET /enseignants - Liste des enseignants
POST /enseignants - Création enseignant
GET /demandes - Liste des demandes
POST /demandes - Création demande
PUT /demandes/{id} - Modification demande
POST /upload - Upload de documents
GET /files/{filename} - Téléchargement fichier
